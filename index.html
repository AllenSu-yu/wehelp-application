Huei-Yu Su的 WeHelp 工程師訓練申請

🧑‍💻 第一題：個人簡介和申請動機

我在 91APP 當 軟體測試工程師(QA)三年，平常會用 Postman 打 API、寫 SQL 查資料，還曾經用 AI 幫忙生出 Mock Service (Python Flask)來加速測試流程。這些經驗讓我開始對 coding 產生興趣，覺得寫程式不只是工具，更是一種解決問題的超能力！現在我想更深入學習程式開發，不管是自動化測試、前端還是後端，只要能讓我技術升級、職涯更有選擇，我都願意投入。希望能透過 WeHelp 的訓練，開啟我成為工程師的下一步！

📚 第二題：曾經做過哪些軟體工程技術相關的學習？若有作品請分享給我們。

除了工作中接觸 Postman API 測試和 SQL，我也開始主動學習程式語言。去年我跟著 YouTube 上的 CodeShiba 程式柴 的 Python 完整教學，學到變數、型別、input、運算符號、if/else 判斷、while 和 for 迴圈等基礎語法，開始能用程式解決一些簡單問題。 最近也完成了 freeCodeCamp 的「Learn HTML by Building a Cat Photo App」課程，對 HTML 的結構和標籤有初步認識。這些學習讓我更確定自己想往工程師方向發展，也期待透過 WeHelp 的訓練，把這些基礎打得更穩，進一步學習前端和後端的實作能力。

⏱️ 第三題：如果參與這個訓練，會怎麼安排學習時間？

由於還在職，我會在下班後每天安排約 3.5 小時的練習時間，週末則各投入 12 小時，總計每週約41.5 小時以上。平日主要進行技術學習與程式實作，週末則用來複習、整理筆記。我會使用 Cursor 來輔助學習，習慣先自己動手寫程式碼，卡住時再請 Cursor 協助找出問題點，這樣能確保我真正理解每個步驟。我也會搭配 google sheet 做進度追蹤，確保每週都有明確目標與成果。希望透過這樣的安排，穩紮穩打地建立開發能力，為轉職工程師做好準備。

🔍 第四題：軟體技術日新月異，如何確定投入的領域是正確有回報的？

以我目前的背景來說，從軟體品質測試(QA)轉向程式開發，是一個自然且有延伸性的選擇。我會參考真實職缺內容與工程師的職涯分享，來判斷哪些技能是有價值的。像是 API、資料庫、前端框架、自動化測試等，都是我目前積極學習的重點。我相信只要持續精進、保持好奇心，技術的變化不會是阻礙，反而是推動我成長的動力。

🌧️ 第五題：請描述一件產生明顯負面情緒的經歷，如何處理該情緒

曾經在公司遇到其他部門的 QA 工程師離職，導致人力短缺，我被詢問是否能協助分擔部分工作。我當下毫不猶豫地答應，並努力完成任務，但在季末績效考評時，這段付出卻沒有被列入績效。主管的回應是，因為我的職等關係，必須做出更具影響力的成果，例如改善整個測試部門或參與跨部門大型專案，才符合績效標準。當下我感到失落與挫折，但我選擇與主管溝通績效的定義與期待，而不是爭取眼前的績效。雖然我不完全認同這樣的標準，但我可以接受，並將這次經驗轉化為動力，開始學習更多技術、提升競爭力，讓自己有能力承接更具影響力的挑戰。

🤖 第七題：如何看待人工智慧的發展對軟體工程師的影響？

我認為人工智慧的發展對軟體工程師來說既是挑戰，也是機會。AI 工具像是 Copilot、Cursor、ChatGPT 等，已經能協助工程師撰寫程式碼、除錯、生成測試案例，甚至優化架構。這讓工程師的工作從「重複勞動」轉向「高層次思考」，更專注在設計邏輯、解決問題與溝通需求上。 對我來說，AI 不會取代工程師，但會淘汰不願學習的人。我希望自己能成為「懂得善用 AI 的工程師」，把它當成加速器而不是競爭者。像我現在學習程式時，就會先自己嘗試，卡住再請 AI 幫忙分析錯誤，這樣不只提升效率，也能強化理解。我相信未來的工程師，會是「懂技術、會溝通、能整合 AI」的全方位角色，而我正在往這個方向努力。
